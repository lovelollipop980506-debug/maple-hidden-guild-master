# Claude Design 프롬프트 — 디자인(시안) 요청 단계

워크플로: ① 이 프롬프트로 **디자인 시안**을 받는다(Claude Design) → ② 그 시안을 Next.js에
넣고 백엔드 API에 연결한다(별도 구현 단계). 이 프롬프트는 ①에만 집중 — 코드/통신 얘기 없음.

---

```
# 역할
너는 시니어 프로덕트 디자이너다. 아래 웹앱의 화면을 디자인(시안)한다. 코드 구현이나 백엔드
연결은 다른 단계에서 하므로, 너는 화면 레이아웃·컴포넌트·상태 디자인에만 집중한다.

# 가장 중요한 제약: 기존 디자인 시스템 사용
- 이미 구축된 디자인 시스템(컬러 토큰, 타이포, 간격, 버튼/카드/배지/입력/테이블/모달/토스트
  등)을 그대로 사용한다. 새 비주얼 언어를 만들지 말고, 시스템에 없으면 기존 스타일에 맞춰
  최소 확장한다.
- 한국어 UI. 데스크톱 우선 + 모바일 반응형.

# 제품
디스코드 길드 관리 웹앱. 길드원은 가입 신청과 스킬 포인트 등 "폼"을 제출하고, 운영자는 이를
검토(승인/반려)하며, 멤버·포인트 관리와 통계를 본다. 로그인은 디스코드 계정.

# 사용자 등급 (화면/메뉴 노출 차등)
admin > reviewer(운영자) > member > guest
- guest: 로그인만 함 — "가입 신청"만 가능
- member: 일반 폼 제출 가능
- reviewer: 검토·멤버·통계
- admin: 폼 관리·설정까지

# 디자인할 화면 (각: 레이아웃 · 핵심 컴포넌트 · 표시 데이터 · 상태[기본/로딩/빈/에러])

1) 로그인
   - 중앙 카드: 서비스 한 줄 소개 + "디스코드로 로그인" 버튼 1개.

2) 대시보드(홈)
   - 내 요약 카드: 등급 배지, 멤버 상태(미신청/신청함/가입됨/반려), 보유 포인트.
   - 바로가기 카드: "가입 신청", "스킬 인증"(member↑만).
   - 최근 내 제출 3~5건(폼 제목/상태 배지/날짜).
   - admin/소유자 & 설정 미완료면 상단에 "초기 설정 필요" 배너.

3) 가입 신청 폼
   - 입력: 캐릭터명(필수), 플레이 시간대, 가입 경로, 자기소개(여러 줄).
   - 제출 버튼, 제출 성공/실패 피드백, "이미 검토 대기 중" 안내 케이스.
   - 하단: 내 이전 신청 내역(상태 배지/사유).

4) 스킬 포인트 인증 폼
   - 상단: 내 보유 포인트.
   - 입력: 스킬/항목(필수), 포인트(숫자, 필수), 설명, 증빙 이미지 업로드(선택, 미리보기).
   - 제출 + 내 인증 내역 리스트(항목/포인트/상태/증빙 썸네일).

5) 내 제출 내역
   - 리스트/테이블: 폼 제목, 제출일, 상태 배지(대기/승인/반려), 반려 사유, 증빙 링크.

6) 검토 큐 (reviewer+)
   - 필터 바: 폼 종류 / 상태(기본=대기) / 소스(웹 제출 · 디스코드 수집).
   - 제출 카드: 제출자, 제출 내용(폼 항목별 값), 이미지 있으면 썸네일.
     · 디스코드 수집 항목은 작성자명 + 메시지 내용 + 이미지로 표시.
   - 각 카드 액션: [승인] [반려] + 사유 입력. 처리 후 상태 변화.

7) 멤버 관리 (reviewer+)
   - 테이블: 멤버명, 등급 배지, 가입 상태, 보유 포인트, 최근 로그인. 정렬(포인트/최근).
   - 행 클릭 → 멤버 상세: 프로필, 포인트 변동 내역(타임라인), admin이면 "포인트 수동 조정"(증감/사유).

8) 통계 (reviewer+)
   - KPI 카드: 전체 멤버, 가입 멤버, 대기 제출, 누적 포인트, 활성 폼.
   - 차트: 포인트 상위 멤버(막대), 제출 상태 분포(도넛).

9) 디스코드 연동 설정 (admin/소유자)
   - 감지된 서버 표시(이름).
   - 알림 채널 선택(드롭다운).
   - 역할 → 등급 매핑 표: 각 역할에 admin/reviewer/member/미지정 선택(추천값 미리 선택됨).
   - 저장 버튼 + "다시 로그인하면 적용" 안내.

# 공통 컴포넌트(디자인 시스템 기반)
상단 내비게이션(좌측 로고, 등급별 메뉴, 우측 사용자+등급배지+로그아웃), 토스트,
상태 배지(대기=주의색/승인=성공색/반려=위험색), 폼 입력(텍스트/숫자/여러줄/선택/체크/파일),
데이터 테이블, 카드, 확인 모달, 빈 상태·로딩·에러 상태.

# 산출물 (순서)
1) 먼저: 위 화면들을 "기존 디자인 시스템의 어떤 컴포넌트로 구성할지" 매핑한 화면·컴포넌트
   인벤토리(텍스트)로 제시.
2) 합의 후: 화면별 고해상도 시안을 데스크톱+모바일, 주요 상태(기본/빈/에러) 포함해 제작.
```
